AMBILUM DATASET DESIGN (FINAL VERSION)

This dataset records environmental conditions, system decisions, and
user behavior in the Ambilum smart lighting system. It is designed to
support both automation logging and future behavioral analysis research.

  -----------------------
  1. CORE SYSTEM FIELDS
  -----------------------

timestamp Exact date and time of the recorded event.

lux Ambient light level measured by the BH1750 sensor.

hour_of_day Hour extracted from the timestamp (0–23).

day_of_week Day of the week (Monday–Sunday).

auto_rgb_profile RGB profile automatically selected by the system.

auto_rgb_brightness RGB brightness automatically selected by the system
(0–100).

auto_monitor_brightness Monitor brightness automatically selected by the
system (0–100).

auto_profile_colors Colors defined inside the automatically selected RGB
profile. Stored as a list of HEX values.

user_override Boolean value indicating whether the user manually changed
system settings.

user_rgb_profile RGB profile manually selected by the user.

user_rgb_brightness RGB brightness manually selected by the user.

user_monitor_brightness Monitor brightness manually selected by the
user.

user_chosen_colors Actual colors chosen manually by the user across
devices. Stored as a list of HEX color values.

override_delay_seconds Time between the automatic system decision and
the user override action.

  ------------------------------
  2. ADDITIONAL CONTEXT FIELDS
  ------------------------------

application_type Category of the active application at the time of the
event. Examples: coding gaming browsing video other

system_mode Current operating mode of the system. Possible values: auto
manual scene

scene_name Name of the active scene if scene mode is active. Examples:
study_mode gaming_mode night_mode movie_mode

  ----------------------------------
  3. RESEARCH STYLE CONTEXT FIELDS
  ----------------------------------

device_id Identifier for the IoT sensor node. Example: desk_node_1

session_id Unique identifier for a continuous PC usage session.

environment_state General lighting condition derived from lux values.
Possible values: dark dim bright

decision_source Indicates how the lighting decision was made. Possible
values: rule_engine user_override

lux_change_rate Rate of change in ambient light level compared to
previous reading.

profile_change_reason Reason the RGB profile changed. Possible values:
lux time application user
